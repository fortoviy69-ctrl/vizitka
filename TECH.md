# Технологический стек: ИИ-ассистент для ведущих «Территории денег»

## Технологии

Сравнительный анализ по 5 категориям для MVP с минимальным бюджетом.

---

## 1. OCR / Распознавание рукописного текста

Задача: принять фото рукописного бланка (8 страниц) и метафорических карт, извлечь текст и структуру.

---

### Claude Vision (claude-sonnet-4-5 / claude-opus-4)

**Плюсы:**
- Понимает контекст, а не только текст: может одновременно распознать рукопись и структурировать её в JSON с полями бланка
- Отлично читает смешанный контент: рукописный текст + печатные заголовки + иконки (монеты, кристаллы)
- Один запрос = OCR + первичная интерпретация, без второго вызова к LLM
- Хорошо справляется с русским рукописным текстом при нормальном качестве фото
- Нет отдельной настройки — просто передаёшь изображение в стандартный API

**Минусы:**
- Платно с первого запроса, нет бесплатного тира
- При плохом освещении или нечётком фото качество падает, как у любого Vision-решения
- Цена выше, чем у специализированных OCR-сервисов

**Цена:** ~$0.003–0.005 за фото (Sonnet), ~$0.015–0.025 за фото (Opus). При 50 сессиях в месяц — $0.15–1.25/мес только на OCR.

---

### GPT-4o Vision

**Плюсы:**
- Сопоставимое качество с Claude Vision на рукописном русском тексте
- Также совмещает OCR и первичную интерпретацию в одном запросе
- Хорошо документирован, огромная база примеров в открытом доступе
- Поддерживает JSON mode — удобно для структурированного вывода

**Минусы:**
- Цена сопоставима с Claude, нет явного преимущества по качеству OCR
- При работе из России — возможны проблемы с оплатой (карты РФ не принимаются напрямую, нужна промежуточная оплата через сервисы-посредники)
- Контекстное окно для Vision запросов — ограничение по количеству изображений за раз

**Цена:** ~$0.00250 за 1K входящих токенов + $0.010 за 1K исходящих. Одно фото бланка (средняя страница) — примерно $0.003–0.008.

---

### Google Cloud Vision API

**Плюсы:**
- Специализированный OCR — быстрее, чем LLM, и дешевле на чистых задачах извлечения текста
- `DOCUMENT_TEXT_DETECTION` — режим специально для плотного текста с разметкой, распознаёт позиции символов
- Стабильное качество на печатном тексте
- 1000 запросов в месяц — бесплатно

**Минусы:**
- Рукописный русский — заметно хуже, чем LLM-based Vision. Google обучал модели преимущественно на латинице
- Не понимает структуру бланка: получишь сырой текст без понимания, что где находится — нужен дополнительный парсинг
- Требует отдельный вызов LLM для интерпретации: итого 2 API-вызова вместо одного
- Настройка GCP — больше времени на старте

**Цена:** $1.50 за 1000 запросов (после бесплатных 1000/мес). При 50 сессиях/мес — практически бесплатно, но качество на рукописи ниже.

---

### Azure AI Vision (Read API)

**Плюсы:**
- Лучший среди облачных OCR-сервисов для рукописного текста, включая кириллицу
- Отдельная модель специально для рукописи (`handwritten` mode)
- Возвращает структуру: строки, слова, bounding boxes
- $200 бесплатного кредита при регистрации

**Минусы:**
- Всё ещё хуже LLM-Vision на понимании контекста и структуры сложного бланка
- Требует отдельный вызов LLM для интерпретации — 2 запроса
- Регистрация Azure из России — сложнее, чем другие варианты
- Документация и поддержка на русском — слабее

**Цена:** $1.50 за 1000 транзакций (S1 tier). 5000 транзакций/мес бесплатно на F0 tier.

---

### Рекомендация по OCR

**Для MVP: Claude Vision.** Один запрос решает две задачи — распознавание и структурирование. Качество на рукописном русском лучше специализированных сервисов. При объёмах MVP (10–100 сессий/мес) стоимость минимальна.

## 2. LLM для анализа и генерации планов

Задача: принять структурированные данные из бланка, проинтерпретировать паттерны, создать 30-дневный план действий на русском языке.

---

### Claude API (Anthropic)

**Плюсы:**
- Лучшее качество русскоязычной генерации среди топ-моделей
- Большое контекстное окно: 200K токенов у Sonnet и Opus — весь бланк + история сессий влезает одним запросом
- Instruction following — строго следует формату вывода (JSON, Markdown, структурированный текст)
- Уже используется в проекте (ты сейчас работаешь с Claude Code)
- Extended thinking (claude-opus-4) — для сложных интерпретаций и нетривиальных паттернов

**Минусы:**
- Нет бесплатного тира в API
- Оплата — карты РФ не принимаются напрямую, нужна виртуальная карта (например, через Privacy.com или зарубежный счёт)

**Цена (claude-sonnet-4-5):** $3 / 1M входящих токенов, $15 / 1M исходящих. Один полный анализ сессии (большой промпт + вывод) — примерно $0.02–0.05. При 50 сессиях/мес — $1–2.50/мес.

---

### OpenAI API (GPT-4o / GPT-4.1)

**Плюсы:**
- Сопоставимое качество по русскому языку
- JSON mode, Structured Outputs — удобно для формирования плана
- Огромная экосистема: больше туториалов, примеров, библиотек
- Function calling хорошо задокументирован

**Минусы:**
- Оплата из России — те же проблемы с картами
- Качество русского чуть уступает Claude на длинных генерациях и нюансированных текстах
- GPT-4o иногда «срезает» детали при следовании сложным инструкциям

**Цена (GPT-4o):** $2.50 / 1M входящих токенов, $10 / 1M исходящих. Сопоставимо с Claude Sonnet.

---

### Yandex GPT (YandexGPT Pro / Lite)

**Плюсы:**
- Оплата рублями, карты РФ работают без проблем
- Русский язык — родной, не переведённый: отлично понимает культурный и языковой контекст
- Работает в российской юрисдикции: данные хранятся в РФ — важно если есть вопросы по персональным данным
- Дешевле в рублёвом эквиваленте при текущем курсе
- Интеграция с Yandex Cloud — базы данных, хостинг, всё в одном экосистеме

**Минусы:**
- Заметно уступает Claude и GPT-4o по качеству сложных рассуждений и следованию инструкциям
- Контекстное окно — 32K токенов у Pro (значительно меньше, чем у Claude/GPT-4o)
- Документация и примеры менее развиты
- При росте проекта придётся мигрировать на более мощные модели

**Цена:** ~0.20 руб / 1000 токенов (Lite), ~2.00 руб / 1000 токенов (Pro). При 50 сессиях/мес — ~100–500 руб/мес.

---

### Рекомендация по LLM

**Для MVP: Claude API (claude-sonnet-4-5).** Лучшее сочетание качества русского языка, контекстного окна и следования инструкциям. Если оплата API проблематична из РФ — **YandexGPT Pro** как fallback с потерей в качестве генерации.

## 3. База данных

Задача: хранить профили ведущих, список клиентов (игроков) каждого ведущего, историю сессий, сформированные планы.

---

### SQLite

**Плюсы:**
- Нулевая стоимость: встроена в Python, не нужен отдельный сервер
- Нулевая настройка: один файл `.db` рядом с кодом
- Отлично подходит для MVP до 100–200 пользователей
- Бэкап = копирование одного файла

**Минусы:**
- Не масштабируется: при конкурентных запросах (несколько ведущих одновременно) начинаются блокировки
- Нет веб-интерфейса для просмотра данных без дополнительных инструментов
- При деплое на Render/Railway — файл БД живёт на эфемерном диске и сбрасывается при рестарте (нужен постоянный Volume)

**Цена:** $0/мес.

---

### PostgreSQL через Supabase

**Плюсы:**
- Бесплатный тир: 500MB хранилища, 2 проекта — хватит на MVP и первые месяцы
- Полноценный PostgreSQL с веб-интерфейсом (Table Editor) — удобно смотреть данные без SQL
- Автоматические бэкапы, репликация
- Готовый REST API (PostgREST) и SDK для Python, JS — меньше кода
- Realtime subscriptions если понадобится в будущем
- Легко масштабируется при росте

**Минусы:**
- Бесплатный проект засыпает после 1 недели инактивности (первый запрос медленный — ~2–5 сек)
- Лимит подключений на бесплатном тире — не критично для MVP
- Данные хранятся на зарубежных серверах — учитывать при работе с персональными данными

**Цена:** Бесплатно до 500MB. Pro план — $25/мес.

---

### Firebase Firestore

**Плюсы:**
- Бесплатный Spark план: 1GB хранилища, 50K чтений/день — достаточно для старта
- Реальтайм обновления из коробки
- Масштабируется автоматически
- Хорошая SDK для Python

**Минусы:**
- NoSQL: для сессионных данных с историей — менее удобен, чем реляционная БД
- Сложнее делать выборки и аналитику (нет JOIN)
- Google экосистема: другой вендор-лок
- Платный уровень — сложная тарификация за операции (платишь за каждый read/write)
- Firebase из России — тот же вопрос с оплатой

**Цена:** Бесплатно (Spark). Pay-as-you-go от $0.06 / 100K reads.

---

### Рекомендация по БД

**Для MVP: SQLite → Supabase.** Начни с SQLite (нулевая сложность), но заложи абстракцию через SQLAlchemy — так миграция на PostgreSQL займёт один день. При первых реальных пользователях сразу переходи на Supabase Free.

## 4. Хостинг / Деплой

Задача: держать Telegram-бот (или веб-приложение) работающим 24/7.

---

### Railway

**Плюсы:**
- Самый удобный деплой: `railway up` или через GitHub — приложение запущено за 2 минуты
- Поддержка Python, Dockerfile без настройки
- Автодеплой при пуше в GitHub
- Постоянный диск (Volume) для SQLite — можно не переходить на внешнюю БД сразу
- Понятная документация, активное комьюнити

**Минусы:**
- Бесплатного тира нет (убрали в 2023): стартовый план от $5/мес + оплата ресурсов
- Реальная стоимость при постоянной работе — $5–15/мес для лёгкого бота
- Оплата — нужна зарубежная карта

**Цена:** $5/мес (Hobby plan) + ~$0.000463/vCPU-мин и $0.000231/GB-мин RAM. Бот с низкой нагрузкой — $5–10/мес.

---

### Render

**Плюсы:**
- Бесплатный тир для Web Services существует (Free plan)
- Похожий на Railway UX: деплой из GitHub одним кликом
- PostgreSQL бесплатно на 90 дней (потом $7/мес)
- Автодеплой по коммиту

**Минусы:**
- Бесплатный тир: сервис засыпает после 15 минут инактивности — первый запрос ждёт ~30–50 сек. Для Telegram-бота это критично (бот не отвечает на первое сообщение)
- Для 24/7 без сна — от $7/мес (Starter plan)
- Медленнее Railway в плане удобства работы

**Цена:** Бесплатно (со сном) или $7/мес (без сна).

---

### Fly.io

**Плюсы:**
- Щедрый бесплатный тир: 3 shared-CPU VM с 256MB RAM бесплатно, без сна
- Postgres бесплатно (до 3GB)
- Поддержка Volume для постоянного хранилища
- Серверы в Европе (Frankfurt, Amsterdam) — низкая задержка для RU пользователей
- Docker-based деплой — максимальная гибкость

**Минусы:**
- Кривая обучения выше: нужен `flyctl`, `fly.toml`, понимание Dockerfile
- Документация хорошая, но UI хуже Railway
- Оплата — нужна зарубежная карта (хотя бесплатный тир не требует карты для старта)

**Цена:** Бесплатно для 1 малого приложения (shared-1x-cpu, 256MB RAM). При росте — pay-as-you-go от $1.94/мес за VM.

---

### VPS у Timeweb / Beget (РФ)

**Плюсы:**
- Оплата рублями, карты РФ — без проблем
- Полный контроль над сервером
- Нет ограничений по ресурсам, кроме оплаченных
- Данные хранятся в РФ — плюс для персональных данных
- Стабильная работа без засыпания
- Timeweb: VPS от 149 руб/мес (1 vCPU, 512MB RAM)

**Минусы:**
- Надо настраивать сервер руками: nginx, systemd, SSL-сертификат, деплой-скрипты
- Нет автодеплоя из коробки — нужно настраивать CI/CD или деплоить вручную
- При проблемах на сервере — разбираться самому
- Время на первичную настройку: 2–4 часа

**Цена:** 149–500 руб/мес (Timeweb), 200–600 руб/мес (Beget).

---

### Рекомендация по хостингу

**Для MVP из России: Fly.io (бесплатный тир) или VPS Timeweb.**
- Fly.io: нулевая стоимость, 24/7 без сна, серверы в Европе. Сложнее в настройке, но один раз разобраться — и всё работает.
- Timeweb VPS: если важно платить рублями и держать данные в РФ. ~200 руб/мес за VPS с ручной настройкой.

## 5. Telegram-бот фреймворк

Задача: принимать фото и текстовые сообщения, управлять сессиями, отправлять структурированные ответы.

---

### python-telegram-bot

**Плюсы:**
- Самая популярная Python-библиотека для Telegram: огромное комьюнити, тысячи туториалов на русском
- Отличная документация с примерами
- Начиная с версии 20.x — полностью async (asyncio)
- ConversationHandler — встроенная поддержка диалогов и состояний
- Стабильная, активно поддерживается

**Минусы:**
- До v20 была синхронной — старые туториалы могут вводить в заблуждение
- Чуть более многословный код по сравнению с aiogram

**Цена:** Бесплатно (open source).

---

### aiogram (Python)

**Плюсы:**
- Разработан в России, документация частично на русском
- Полностью async с самого начала — лучшая производительность при нагрузке
- Современный, pythonic API: декораторы, роутеры, middleware
- FSM (Finite State Machine) — встроенная машина состояний для управления диалогом
- Активное развитие, aiogram 3.x — современный стандарт для production-ботов

**Минусы:**
- Кривая обучения чуть выше python-telegram-bot
- Документация v3 писалась в спешке, некоторые разделы неполные

**Цена:** Бесплатно (open source).

---

### Telegraf (Node.js)

**Плюсы:**
- Лучший фреймворк для Telegram на JavaScript/TypeScript
- Middleware-архитектура (как Express.js) — интуитивна для JS-разработчиков
- TypeScript-first — хорошая типизация

**Минусы:**
- Node.js, а не Python: если остальной стек на Python (Claude/OpenAI SDK, SQLAlchemy) — лишний язык в проекте
- Смешивать Python-бэкенд и Node.js-бот = усложнение архитектуры
- Для проекта с упором на ИИ-часть Python предпочтительнее: все AI SDK лучше поддерживаются в Python

**Цена:** Бесплатно (open source).

---

### Рекомендация по фреймворку

**Для MVP: aiogram 3.x.** Современный async Python, встроенный FSM для управления диалогом (выбор: фото или текст? подтверждение? итд), активная поддержка. Отличная синергия с остальным Python-стеком (Anthropic SDK, SQLAlchemy).

### Архитектура MVP одной строкой

```
Telegram (aiogram) → Claude Vision (OCR + структурирование бланка) → Claude API (анализ + план) → SQLite/Supabase → ответ ведущему
```

### Технический долг, который закладывается сразу

- SQLAlchemy как ORM — чтобы можно было сменить SQLite на PostgreSQL без переписывания кода
- Конфигурация через `.env` файл — API-ключи не в коде
- Отдельный модуль для LLM-провайдера — чтобы можно было сменить Claude на OpenAI/YandexGPT без переписывания логики
